# AW-2349 Phase 7: DI Wiring + `screenLocked` Event

Status: PLAN_APPROVED

## Phase Scope

Phase 7 wires `ScreenLockService` (created in Phase 6) through the example app's factory-based DI graph and declares the `LockerEvent.screenLocked()` Freezed event. Phase 8 will add the actual `LockerBloc` handler and pass the service into the BLoC constructor.

Specifically, this phase:
1. Introduces a `BiometricCipher` field in `RepositoryFactoryImpl` and uses it to create `ScreenLockServiceImpl`.
2. Exposes `ScreenLockService` via a getter on the `RepositoryFactory` abstract interface.
3. Adds `ScreenLockService` as a constructor parameter and field to `BlocFactoryImpl` (stored but NOT forwarded to `LockerBloc` -- deferred to Phase 8).
4. Wires `repositoryFactory.screenLockService` into `BlocFactoryImpl` construction in `main.dart`.
5. Declares `LockerEvent.screenLocked()` as a zero-argument Freezed factory.
6. Runs code generation to regenerate Freezed output.

This phase modifies 4 source files and 1 generated file. No tests are added in this phase.

## Components

### 1. `RepositoryFactory` / `RepositoryFactoryImpl`

File: `example/lib/di/factories/repository_factory.dart`

Changes:
- Add `import 'package:biometric_cipher/biometric_cipher.dart'` and `import 'package:mfa_demo/core/services/screen_lock_service.dart'`.
- Add `ScreenLockService get screenLockService;` to the abstract `RepositoryFactory` interface.
- Add `late final BiometricCipher _biometricCipher;` field to `RepositoryFactoryImpl`.
- Add `late final ScreenLockService _screenLockService;` field to `RepositoryFactoryImpl`.
- Add getter: `ScreenLockService get screenLockService => _screenLockService;`.
- In `init()`, before `configureBiometricCipher(...)`: instantiate `_biometricCipher = BiometricCipher()` and `_screenLockService = ScreenLockServiceImpl(biometricCipher: _biometricCipher)`.
- In `dispose()`, before `await _lockerRepository?.dispose()`: call `_screenLockService.dispose()`.

Design decisions:
- `late final` rather than nullable (`?`): The PRD specifies `late final` for the new fields. This differs from the existing `_timerService` which uses nullable + `StateError` guard. `late final` is acceptable because `init()` is always called before the getter is accessed, and a `LateInitializationError` is a clear signal if that contract is violated.
- Initialization order in `init()`: `_timerService` (existing) -> `_biometricCipher` (new) -> `_screenLockService` (new) -> `configureBiometricCipher(...)` (existing). The `_biometricCipher` must be set before `_screenLockService` since it is a constructor dependency.
- Disposal order: `_screenLockService.dispose()` (synchronous) is called before `await _lockerRepository?.dispose()` (async). The `_timerService?.dispose()` call remains first.

### 2. `BlocFactory` / `BlocFactoryImpl`

File: `example/lib/di/factories/bloc_factory.dart`

Changes:
- Add `import 'package:mfa_demo/core/services/screen_lock_service.dart'`.
- Add `required ScreenLockService screenLockService` as a constructor parameter (after `timerService`).
- Add `final ScreenLockService _screenLockService;` field (after `_timerService`).
- Add `_screenLockService = screenLockService` to the initializer list (after `_timerService = timerService`).
- Remove `const` keyword from the constructor declaration.
- Do NOT pass `_screenLockService` to `LockerBloc(...)` in the `lockerBloc` getter. `LockerBloc` does not accept this parameter yet -- that is Phase 8 scope.

Design decisions:
- `const` removal: The current constructor is `const BlocFactoryImpl({...})`. Adding `ScreenLockService` (an abstract interface) does not prevent `const` in principle, but the existing codebase never constructs `BlocFactoryImpl` with `const` at any call site (only `main.dart` constructs it, without `const`). Removing `const` from the declaration is safe with zero breakage.
- Parameter ordering: `ScreenLockService` is placed after `TimerService` in both the parameter list and field declarations, following the existing ordering convention.

### 3. `main.dart`

File: `example/lib/main.dart`

Changes:
- Add `screenLockService: repositoryFactory.screenLockService` to the `BlocFactoryImpl(...)` constructor call on line 39-42.

No new imports needed -- `BlocFactoryImpl` is already imported, and the parameter is typed by the factory.

### 4. `LockerEvent`

File: `example/lib/features/locker/bloc/locker_event.dart`

Changes:
- Add `const factory LockerEvent.screenLocked() = _ScreenLocked;` as the last event in the sealed class, before the closing `}`.
- Add a doc comment: `/// Device screen was locked. Triggers immediate locker lock.`

This is a zero-argument Freezed factory following the same style as `LockerEvent.lockRequested()`, `LockerEvent.appResumed()`, and `LockerEvent.activityDetected()`.

## Data Flows

### App Startup (Phase 7 contribution)

```
main()
  -> RepositoryFactoryImpl(storageFilePath)
  -> repositoryFactory.init()
       -> _timerService = TimerServiceImpl(...)         [existing]
       -> _biometricCipher = BiometricCipher()          [new]
       -> _screenLockService = ScreenLockServiceImpl(   [new]
            biometricCipher: _biometricCipher)
       -> configureBiometricCipher(...)                 [existing]
  -> BlocFactoryImpl(
       lockerRepository: ...,
       timerService: ...,
       screenLockService: repositoryFactory.screenLockService,  [new]
     )
  -> BlocFactoryImpl stores _screenLockService (unused until Phase 8)
```

### App Teardown (Phase 7 contribution)

```
repositoryFactory.dispose()
  -> _timerService?.dispose()          [existing]
  -> _screenLockService.dispose()      [new - synchronous]
  -> await _lockerRepository?.dispose() [existing]
```

```

  -> Code-generated in locker_bloc.freezed.dart
  -> Available for Phase 8 handler registration
```
